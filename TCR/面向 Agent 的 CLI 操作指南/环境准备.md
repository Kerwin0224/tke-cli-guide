# 环境准备

> 页型 **T0** · TCR Agent 工具链入口 · 不含映射页 page_id

完成以下步骤后，即可运行本指南下的任何 **`tccli 操作`** 任务页。

---

## 1. 安装 tccli

`tccli` 是腾讯云 CLI 工具（需要 Python ≥ 3.8），本指南所有控制面操作均通过它完成。

#### 推荐：使用 uv（跨平台统一管理）

[uv](https://docs.astral.sh/uv/) 统一管理 Python 版本和工具安装，**macOS / Windows / Linux 命令完全一致**：

```bash
# 安装 uv（所有平台）
# macOS / Linux：
curl -LsSf https://astral.sh/uv/install.sh | sh
# Windows（PowerShell）：
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

# 安装 Python 3.12（所有平台，命令一致）
uv python install 3.12

# 安装 tccli（所有平台，命令一致）
uv tool install tccli
```

> `uv tool install` 自动将 `tccli` 加入 PATH。如遇到 `tccli: command not found`，见 [TKE 环境准备 — 故障排除](../../TKE/面向%20Agent%20的%20CLI%20操作指南/环境准备.md#8-故障排除)。

#### 备选：使用系统 Python + venv

<details>
<summary>macOS / Linux</summary>

```bash
python3 -m venv ~/.tccli-venv
source ~/.tccli-venv/bin/activate
pip install tccli
```

</details>

<details>
<summary>Windows</summary>

```cmd
python -m venv %USERPROFILE%\.tccli-venv
%USERPROFILE%\.tccli-venv\Scripts\activate
pip install tccli
```

</details>

### 验证安装

```bash
tccli --version
# 期望输出：3.1.xxx.x
```

## 2. 配置凭证与地域

详细说明（环境变量 / 配置文件 / 优先级）见 [TKE 环境准备 — 配置凭证与地域](../../TKE/面向%20Agent%20的%20CLI%20操作指南/环境准备.md#2-配置凭证与地域)。

快速参考：

```bash
# 环境变量方式（推荐用于 Agent / CI）
export TENCENTCLOUD_SECRET_ID="<YourSecretId>"
export TENCENTCLOUD_SECRET_KEY="<YourSecretKey>"
export TENCENTCLOUD_REGION="ap-guangzhou"

# 或交互式输入
tccli configure
```

## 3. 验证凭证

```bash
# 简单读取验证（不会产生费用）
tccli tcr DescribeInstances --limit 1 --output json
```

成功时返回类似：

```json
{
    "TotalCount": 0,
    "Registries": []
}
```

如遇到鉴权错误，请检查凭证是否正确设置。

## 4. API 入参发现（写命令前必读）

tccli 操作.md 中的每个 `tccli` 命令的 API 参数名必须来自 **API 入参真源**。详细流程（`help` → `help --detail` → `--generate-cli-skeleton` fallback）见 [TKE 环境准备 — API 入参发现](../../TKE/面向%20Agent%20的%20CLI%20操作指南/环境准备.md#4-api-入参发现写命令前必读)。

TCR 快速参考：

```bash
# 查看所有 TCR Action
tccli tcr help

# 查看 Action 参数（macOS / Linux）
tccli tcr CreateInstance help --detail

# Windows GBK 解码失败时的 fallback
tccli tcr CreateInstance --generate-cli-skeleton --output json
```
