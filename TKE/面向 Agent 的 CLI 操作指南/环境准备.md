# 环境准备

> 页型 **T0** · TKE Agent 工具链入口 · 不含映射页 page_id

Complete these steps once before running any **`tccli 操作`** task page under this guide.

## Install tccli

Install the Tencent Cloud CLI per the official guide:

- [安装与配置 tccli](https://cloud.tencent.com/document/product/440/34066)

Verify:

```bash
tccli --version
```

## Configure credentials and region

Set credentials and default region (replace placeholders):

```bash
tccli configure set secretId <SecretId> secretKey <SecretKey> region ap-guangzhou
```

Or use interactive setup:

```bash
tccli configure
```

### Credential priority

When the same setting appears in multiple places, tccli resolves in this order:

1. **Command-line flags** (highest)
2. **`~/.tccli/default.configure`** (profile file)
3. **Environment variables** (`TENCENTCLOUD_SECRET_ID`, `TENCENTCLOUD_SECRET_KEY`, `TENCENTCLOUD_REGION`, …)

Task pages assume `--region` matches the region where your VPC and cluster resources live.

## VPC preflight (cross-service)

Many TKE create flows require an existing VPC and subnet. Preflight in the target region before complex writes:

```bash
tccli vpc DescribeVpcs --region ap-guangzhou --output json
tccli vpc DescribeSubnets --region ap-guangzhou --output json
```

Copy real `VpcId` and `SubnetId` values into curated JSON under `examples/` (see [tccli 专页（TKE）](./tccli 专页（TKE）.md)).
